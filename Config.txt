Fase 0
Vas a actuar como un ingeniero backend senior especializado en Spring Boot, trabajando bajo una arquitectura estrictamente dividida en capas (Controller, Service, Repository, Entity, Config, Security).

En este primer intercambio SOLO necesito la configuración base del proyecto, nada de lógica de negocio todavía. Necesito:

- pom.xml con las dependencias necesarias para: Spring Web, Spring Data JPA, Spring Security, PostgreSQL driver, Lombok, Flyway, y la librería jjwt en su versión 0.12.x (jjwt-api, jjwt-impl y jjwt-jackson, estas dos últimas con scope runtime).
- application.yml base, más application-local.yml y application-prod.yml como perfiles separados. El perfil activo debe resolverse con spring.profiles.active: ${SPRING_PROFILES_ACTIVE:local}.
- spring.jpa.hibernate.ddl-auto debe quedar en "none" en todos los perfiles, porque el control del esquema lo va a llevar Flyway, no Hibernate.
- En producción, cualquier dato sensible (URL de base de datos, usuario, contraseña, secreto JWT) debe leerse desde variables de entorno, nunca quedar escrito directamente en el YAML.

Reglas que debes respetar en todo momento durante este proyecto:
- No agregues dependencias ni configuraciones que no te haya pedido.
- No uses arquitecturas orientadas a vistas (nada de Thymeleaf, MVC con vistas, etc.) — esto es una API REST pura.
- Si algo no queda claro (nombre de la base de datos, versión de Java, puerto, etc.), pregúntame antes de asumirlo.
- Al entregar cada archivo, indica entre paréntesis a qué capa pertenece, por ejemplo: application.yml (Config).

Fase 2
Continuamos con el mismo proyecto. Ahora necesito el núcleo de seguridad basado en JWT, sin autenticación básica de ningún tipo.

Componentes que necesito, todos en la capa Security:

1. JwtAuth: implementa AuthenticationEntryPoint. Cuando ocurre un error de autenticación, debe responder con un error HTTP 401 usando response.sendError, pasando el mensaje de la excepción.

2. JwtTokenProvider: componente que lee desde application.yml el secreto JWT y el tiempo de expiración usando @Value. Debe exponer:
   - Un método privado para construir la clave de firma a partir del secreto en Base64 (usando Keys.hmacShaKeyFor).
   - Un método para generar el token a partir de un objeto Authentication, usando el builder de Jwts (setSubject, setIssuedAt, setExpiration, signWith).
   - Un método para extraer el username desde el token.
   - Un método para validar el token.

   Importante: usa la API de la versión 0.12.x de jjwt, con la sintaxis Jwts.parser().setSigningKey(...).build()... No la reemplaces por otras variantes de la librería aunque conozcas versiones más nuevas o antiguas — necesito exactamente ese estilo de llamada porque el resto del proyecto depende de que compile con esa versión específica.

3. JwtAuthFilter: filtro que extiende OncePerRequestFilter. Debe extraer el token del header Authorization (formato Bearer), validarlo, cargar el usuario correspondiente y dejarlo en el contexto de seguridad de Spring.

Reglas: respeta los nombres exactos de las clases que te di arriba, indica la capa de cada archivo, y cierra la fase con un resumen antes de seguir a la siguiente.
Fase 3
Seguimos en el mismo proyecto. Ahora arma el flujo completo de login.

Necesito:

1. DTO LoginRequest: con username y password.
2. DTO JwtAuthResponse: con accessToken y un tokenType que por defecto valga "Bearer".
3. Interfaz AuthService con un método login que reciba un LoginRequest y devuelva el token como String.
4. AuthServiceImpl: implementa AuthService. Debe usar AuthenticationManager para autenticar las credenciales, guardar la autenticación en el contexto de seguridad, y luego usar JwtTokenProvider para generar el token.
5. AuthController: expone POST /api/auth/login, recibe un LoginRequest y responde con un JwtAuthResponse.
6. La clase SecurityConfiguration completa: debe deshabilitar CSRF, permitir sin autenticación todo lo que empiece con /api/auth/, exigir autenticación en el resto de rutas, registrar el manejador de errores de autenticación (JwtAuth) y agregar el JwtAuthFilter antes del filtro estándar de usuario/contraseña de Spring. También necesito los beans de PasswordEncoder (BCrypt) y AuthenticationManager.

No uses httpBasic en ningún punto de la configuración — toda la autenticación pasa por JWT.

Fase 5

Mapea el puerto de PostgreSQL a uno distinto del estándar en el host (por ejemplo 5433:5432) para evitar conflictos si ya tengo Postgres corriendo localmente.
